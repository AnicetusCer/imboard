# Imboard name and branding

“Imboard” means “Input Method Board” and is used as the name of the official
upstream application.

The Imboard source code is licensed under GPL-3.0-or-later. That licence allows
people to use, study, share, and modify the software under its terms.

The Imboard name, icon, logo, and other project branding identify the official
project and are not a blanket permission for unofficial releases to present
themselves as official Imboard builds.

## Permitted use

You may use the Imboard name to:

- refer to the official project
- link to the official repository
- describe compatibility, for example "works with Imboard"
- submit patches, issues, documentation, packaging work, or other contributions

## Modified versions

Modified redistributions must not imply that they are official Imboard releases
unless the project maintainer has approved that use.

If you redistribute a modified build, use a clearly distinct name and icon, or
make the modification status prominent enough that users cannot confuse it with
the official Imboard project.

Examples of acceptable descriptions:

- "based on Imboard"
- "forked from Imboard"
- "unofficial build of Imboard with local changes"

Examples of unacceptable descriptions:

- presenting a modified build as the official Imboard release
- using the official Imboard icon for a materially changed downstream build
- using the Imboard name in a way that suggests maintainer endorsement

## Package metadata

Distribution packages of unmodified official source releases may use the Imboard
name and icon. Packages carrying downstream patches should disclose those
patches in the package metadata or changelog.

This policy is intended to protect users from confusing unofficial builds while
keeping normal open-source contribution and packaging workflows possible.
